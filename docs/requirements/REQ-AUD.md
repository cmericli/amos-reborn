## Paula Register Interface

```
REQ-AUD-001: Paula channels -- 4 channels with per-channel state
  Acceptance: paula_init creates paula_t with exactly 4 paula_channel_t entries (channels[0..3]);
              each channel has: sample_data pointer, sample_length, period, volume, position, active flag, envelope
  Level: 1 (unit)
  Source: Blueprint S4 "Paula Register Interface"
  Status: IMPL
```

```
REQ-AUD-002: Paula base addresses -- channels map to Amiga chip registers
  Acceptance: Channel 0 = $DFF0A0 (MuChip0), Channel 1 = $DFF0B0 (MuChip1),
              Channel 2 = $DFF0C0 (MuChip2), Channel 3 = $DFF0D0 (MuChip3);
              16-byte spacing between channels verified in any hardware-mapping layer
  Level: 1 (unit)
  Source: Blueprint S4 "Paula Register Interface"
  Status: TODO -- no Amiga address constants defined; emulation uses array index only
```

```
REQ-AUD-003: Per-channel registers -- correct offsets and semantics
  Acceptance: Each channel exposes: +$00 = sample pointer (longword),
              +$04 = length in words (word), +$06 = period (word), +$08 = volume 0-64 (word);
              paula_channel_t fields map to these: sample_data (+$00), sample_length (+$04),
              period (+$06), volume (+$08, clamped 0-64)
  Level: 1 (unit)
  Source: Blueprint S4 "Paula Register Interface"
  Status: IMPL -- fields exist; sample_length stored as bytes not words (cosmetic difference, functionally correct)
```

```
REQ-AUD-004: PAL clock constant -- 3,546,895 Hz
  Acceptance: PAULA_PAL_CLOCK == 3546895; frequency = PAULA_PAL_CLOCK / (period * 2)
  Level: 1 (unit)
  Source: Blueprint S4 "Clock"
  Status: IMPL -- #define PAULA_PAL_CLOCK 3546895 in amos.h; freq formula in paula.c line 72
```

```
REQ-AUD-005: NTSC clock constant -- 3,579,545 Hz
  Acceptance: PAULA_NTSC_CLOCK == 3579545; selectable via runtime flag or compile option
  Level: 1 (unit)
  Source: Blueprint S4 "Clock"
  Status: TODO -- no NTSC constant defined anywhere in codebase
```

```
REQ-AUD-006: DMA wait -- 5 raster lines after toggling
  Acceptance: After enabling/disabling channel DMA, system waits >= 5 raster line equivalents
              before accessing channel registers; original polls $DFF006;
              emulation must introduce equivalent delay (~5 * 63.5us = ~317.5us at PAL timing)
  Level: 2 (subsystem)
  Source: Blueprint S4 "DMA Wait"
  Status: TODO -- no DMA wait logic implemented; channels activate immediately
```

```
REQ-AUD-007: Sample bank header -- word count at offset 0

```

```
REQ-AUD-008: Sample bank offset table -- longword per sample
  Acceptance: After the sample count word, bank contains one 32-bit longword offset per sample,
              pointing to the start of each sample's header within the bank
  Level: 1 (unit)
  Source: Blueprint S4 "Sample Format (Bank 5)"
  Status: TODO
```

```
REQ-AUD-009: Per-sample header format -- 6+2+2+4 bytes then PCM data
  Acceptance: Each sample entry contains: 6-byte name (null-padded ASCII),
              2 bytes reserved, 2-byte frequency in Hz (big-endian),
              4-byte length in bytes (big-endian), followed by raw 8-bit signed PCM data
  Level: 1 (unit)
  Source: Blueprint S4 "Sample Format (Bank 5)"
  Status: TODO
```

```
REQ-AUD-010: Sample data format -- 8-bit signed PCM
  Acceptance: Sample data is raw 8-bit signed (-128 to +127) PCM, no header/compression;
              paula playback treats sample_data as int8_t array
  Level: 1 (unit)
  Source: Blueprint S4 "Sample Format (Bank 5)"
  Status: IMPL -- paula.c reads sample_data as int8_t (line 115: c->sample_data[pos])
```

## Built-in Waveforms

```
REQ-AUD-011: Wave 0 -- LFSR noise with multiplier $3171
  Acceptance: 256-byte noise waveform generated by: seed starts non-zero,
              seed = seed * 0x3171 (uint16_t multiply), byte[i] = (int8_t)(seed >> 8);
              output matches reference LFSR sequence exactly
  Level: 1 (unit)
  Source: Blueprint S4 "Built-in Sound Effects"
  Status: IMPL -- effects.c lines 50-54, seed=1, multiplier=0x3171
```

```
REQ-AUD-012: Wave 1 -- square wave 127/-127
  Acceptance: 256-byte square waveform: bytes 0-127 = +127, bytes 128-255 = -127
  Level: 1 (unit)
  Source: Blueprint S4 "Built-in Sound Effects"
  Status: IMPL -- effects.c lines 69-72
```

```
REQ-AUD-013: Sub-octave decimation -- 8 levels by averaging pairs
  Acceptance: From base 256-byte waveform, generate 7 sub-octaves: 128, 64, 32, 16, 8, 4, 2 bytes;
              each level computed by averaging adjacent pairs from previous level:
              sub[i] = (prev[i*2] + prev[i*2+1]) / 2;
              total storage = 256+128+64+32+16+8+4+2 = 510 bytes per wave
  Level: 1 (unit)
  Source: Blueprint S4 "Built-in Sound Effects"
  Status: IMPL -- effects.c lines 57-66 (noise) and 75-84 (square), WAVE_TOTAL_SIZE=510
```

## Stereo Assignment

```
REQ-AUD-019: Amiga stereo panning -- ch 0+3 left, ch 1+2 right
  Acceptance: During mixing, channels 0 and 3 contribute to left output only;
              channels 1 and 2 contribute to right output only;
              no crossfeed between sides (hard pan, authentic Amiga behavior)
  Level: 2 (subsystem)
  Source: Blueprint S4 "Built-in Sound Effects" / paula.c header comment
  Status: IMPL -- paula.c lines 119-123: if (ch==0||ch==3) left += sample; else right += sample;
```

## Envelope System

```
REQ-AUD-020: Envelope format -- 16.16 fixed-point linear interpolation
  Acceptance: Envelope volume uses fixed-point arithmetic (or equivalent double precision);
              per VBL tick: current_vol += delta; delta = (target - current) / duration;
              volume snaps to target at end of each segment
  Level: 1 (unit)
  Source: Blueprint S4 "Envelope System"
  Status: IMPL -- effects.c uses double (equivalent precision); amos_envelope_tick() line 163:
          env->current_vol += env->delta; snap at line 168: env->current_vol = env->target_vol
```

```
REQ-AUD-021: Envelope processing rate -- VBL (50 Hz PAL)
  Acceptance: Envelopes advance once per VBL frame; at PAL rate = 50 Hz;
              paula_mix processes envelope ticks at intervals of output_rate/50 samples
  Level: 2 (subsystem)
  Source: Blueprint S4 "Envelope System"
  Status: IMPL -- paula.c ENVELOPE_RATE=50, envelope_counter triggers process_envelopes()
```

```
REQ-AUD-022: Envelope per-channel independence
  Acceptance: Each of the 4 channels has its own envelope_t instance;
              envelopes advance independently; one channel finishing does not affect others
  Level: 1 (unit)
  Source: Blueprint S4 "Envelope System"
  Status: IMPL -- paula_channel_t contains envelope_t; process_envelopes() loops ch 0..3
```

```
REQ-AUD-023: Envelope volume applied to channel
  Acceptance: After each envelope tick, envelope current_vol (rounded to nearest int, clamped 0-64)
              is written to channel volume; channel deactivates when envelope finishes and vol == 0
  Level: 1 (unit)
  Source: Blueprint S4 "Envelope System"
  Status: IMPL -- paula.c lines 46-54
```

```
REQ-AUD-024: Envelope loop -- -1 value loops to start
  Acceptance: If envelope definition contains -1 as a duration value,
              envelope resets phase to 0 and replays from beginning (infinite loop);
              used for sustained instrument tones
  Level: 1 (unit)
  Source: Blueprint S4 "Envelope System"
  Status: TODO -- amos_envelope_tick() does not check for -1 loop sentinel
```

```
REQ-AUD-025: Envelope terminator -- 0,0 ends envelope
  Acceptance: Duration value of 0 in envelope definition terminates the envelope;
              env->active set to false; standard terminator pair is {0, 0}
  Level: 1 (unit)
  Source: Blueprint S4 "Envelope System"
  Status: IMPL -- effects.c lines 180-184: if (dur == 0) { env->active = false; return; }
```

## Audio Mixer Integration

```
REQ-AUD-034: Audio mixer -- Paula + tracker combined output
  Acceptance: amos_audio_mix() mixes Paula channel output with tracker module output
              into a single stereo int16_t buffer; Paula rendered first, tracker added on top
  Level: 2 (subsystem)
  Source: Blueprint S4 (overall audio architecture)
  Status: IMPL -- mixer.c calls amos_paula_mix() then amos_tracker_mix()
```

```
REQ-AUD-035: Low-pass filter -- A500 RC filter emulation
  Acceptance: Paula output optionally passes through a low-pass filter at ~4400 Hz cutoff;
              filter_enabled flag controls activation; filter is on by default (A500 behavior);
              single-pole IIR approximation of Amiga RC filter
  Level: 2 (subsystem)
  Source: Blueprint S4 / paula.c implementation
  Status: IMPL -- paula.c lowpass() at 4400 Hz, filter_enabled=true by default
```
